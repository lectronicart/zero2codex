# Research and Insights

## Confirmed Facts

### Topic: Git learning safety
**Key insight:** Beginners can learn the correct working-directory, staging,
commit, branch, merge, remote, and pull-request mental models through a
browser-safe Git simulator before using a real repository or GitHub account.

**Why it matters:** Git becomes a confidence and recovery layer around Codex
work instead of another risky setup requirement.

**Source or origin:** Completed Level 4 implementation and simulator tests.

**Date added:** 2026-06-24

**Possible action:** Reuse Git snapshots and diffs in future Codex review,
branch, and worktree lessons.

---

### Topic: Level 1 learning role
**Key insight:** Level 1 is the learner confidence layer. It should explain files, folders, paths, file types, programs, and terminals through browser-safe simulations without requiring a real terminal, local files, an account, or device access.

**Why it matters:** Removing intimidation before command practice prepares true beginners for Level 2 without pretending that computer systems are magical or unknowable.

**Source or origin:** Level 1 implementation brief and completed repository slice.

**Date added:** 2026-06-23

**Possible action:** Reuse the same calm, simulation-first pattern for future prerequisite concepts.

---

### Topic: zero2codex product architecture
**Key insight:** The zero2codex build guide recommends a content-driven Vite + React + TypeScript app with a typed lesson schema, localStorage progress for MVP, browser-safe simulations, and later account sync, community, and admin layers.

**Why it matters:** It creates a practical sequencing strategy: build reusable learning infrastructure before expanding into platform complexity.

**Source or origin:** ZERO2CODEX_BUILD_GUIDE.html, project file.

**Date added:** 2026-06-23

**Possible action:** Use the guide as the source document for Phase 0 and Phase 1 implementation prompts.

---

### Topic: zero2codex MVP scope
**Key insight:** The guide recommends an MVP with Levels 1 and 2 complete, a working terminal simulator, and a five-lesson Level 8 Codex teaser rather than all 151 lessons.

**Why it matters:** This reduces development risk while still demonstrating the course’s learning promise.

**Source or origin:** ZERO2CODEX_BUILD_GUIDE.html, project file.

**Date added:** 2026-06-23

**Possible action:** Treat this as the default launch scope unless user research provides a better sequence.

## Hypotheses and Strategic Opinions

### Topic: Differentiated audience position
**Key insight:** A creator-first angle focused on AI art, video, content systems, digital products, and simple automation may be more differentiated than a broad AI automation school.

**Why it matters:** Strong positioning affects the content strategy, product design, community language, and acquisition channels.

**Source or origin:** Project strategy discussion.

**Date added:** 2026-06-10

**Possible action:** Test messaging with future landing-page copy, social content, and creator interviews.

---

### Topic: zero2codex as a trust asset
**Key insight:** A free, highly practical Codex course may function as a trust-building asset that supports later offers such as templates, workshops, cohorts, and setup support.

**Why it matters:** It gives the business a product-led acquisition path rather than relying entirely on daily social content.

**Source or origin:** Build guide and project strategy discussion.

**Date added:** 2026-06-23

**Possible action:** Define the future paid-support ladder only after the free learning loop earns user proof.
