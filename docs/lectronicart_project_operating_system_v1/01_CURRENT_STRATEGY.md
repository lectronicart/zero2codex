# Current Strategy

## Current Strategy
LectronicArt Skool HQ is being built as a creator-first ecosystem around the promise:

> Build your AI-powered creative machine.

The core flywheel is:

> Create → Teach → Attract → Community → Products → Proof → More Creation

The brand teaches creative builders how to use AI for art, video, content systems, digital products, and simple automations. zero2codex is a complementary product workstream: a free, gamified, Codex-native course that takes beginners from basic computer concepts to safely building and shipping software with Codex.

## Top Priorities
1. Preserve the core LectronicArt positioning and flywheel as the business foundation.
2. Implement and validate the complete Level 1 confidence layer before adding account complexity. Completed 2026-06-23.
3. Continue the sequential zero2codex curriculum after completing the Level 4
   Git and history layer on 2026-06-24.
4. Create durable living documents so strategy, decisions, prompts, and next actions do not get lost across chats.
5. Build the smallest credible zero2codex MVP before expanding into the full 17-level course.

## What We Are Intentionally Not Doing
- Building a generic AI automation agency curriculum
- Attempting all 151 zero2codex lessons before a working MVP exists
- Building accounts, dashboards, forums, maps, admin systems, or full CMS before the core lesson engine proves useful
- Treating Codex as a magic button that removes the need for review, Git, testing, context, or safety
- Copying zero2claude lesson text, exact visuals, code, or private implementation

## Active Experiments
- Designing a 17-level, 151-lesson zero2codex curriculum
- Writing Goal Mode implementation prompts for each major level
- Using a browser-safe simulation strategy for terminal, Git, HTTP, Codex, MCP, and advanced workflows
- Reusing the Level 4 Git simulator for future Codex diff, branch, review, and
  worktree lessons
- Exploring LectronicArt Skool as a creator-first community and product engine

## Risks and Assumptions
### Risks
- The curriculum could become too large before the MVP proves demand.
- Codex product details may change, making some lesson content stale.
- Building interactive simulations can consume substantial product-development time.
- The business could drift away from the creator-first position into generic technical education.

### Assumptions
- Beginners will value a practical path from files and terminal basics to safe Codex collaboration.
- A free, high-quality course can become a trust-building asset for future paid templates, workshops, support, and community offers.
- The initial MVP should focus on Level 1, Level 2, a working terminal simulator, and a short Level 8 Codex teaser.

## Decisions Still Needed
- Whether zero2codex will live under LectronicArt branding, as a standalone sub-brand, or both
- Exact MVP release scope and delivery date
- Primary platform architecture and hosting choices once implementation begins
- Whether the first monetization layer is Skool membership, digital templates, live workshops, or a combination
- Whether account creation and Google login belong in early MVP or after core course validation
